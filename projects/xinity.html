Xinity outreach system

The system that fills Xinity's hackathons, plus the Firebase platform that runs them — registration, team coordination, live dashboard.

The problem

Xinity needed to fill three hackathons and had no budget for outreach tooling.

What I built

The send system that fills the hackathons, plus the Firebase platform that runs them — registration, team coordination, and a live dashboard. Built solo as a summer intern; promoted to Head of Technical Department in two months.

The hardest decision

Instead of paying for a mail service, I built the send system myself. Contacts import from a spreadsheet, one draft is composed once, then delivered in small batches from real accounts rather than blasted from a single bulk sender — so no recipient list ever looked like a campaign. For WhatsApp I split the message queue across the team, so every message went out from a real person's account instead of an API.

Security patch

Found an unauthenticated Firebase API endpoint exposing full event data — every registrant, every team. Patched it independently, within hours, with Firestore security rules. Nobody expects a first-year student to find this; finding it is the point.

The outcome

10,000+ emails and 1,200+ WhatsApp messages delivered without a paid service — ₹0 spent. The platform ran 3+ events including Biothon 2026, 200+ participants.
